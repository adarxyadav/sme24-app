## Coding Behavior (Karpathy)

**Think first.** Surface assumptions before coding. If multiple interpretations exist, name them — don't pick silently. If something is unclear, stop and ask.

**Simplicity first.** Minimum code that solves the problem. No speculative features, no single-use abstractions, no flexibility that wasn't asked for. If 200 lines could be 50, rewrite it.

**Surgical changes.** Touch only what the task requires. Don't improve adjacent code, reformat, or refactor things that aren't broken. Match existing style. If you notice unrelated dead code, mention it — don't delete it. Remove only imports/variables YOUR changes made unused.

**Define success.** Before multi-step tasks, state a brief plan with a verifiable check per step. Transform vague tasks into testable goals before starting.
