# SME24

## Overview

SME24 is an AI-powered EHS (Environmental, Health & Safety) consulting
marketplace that connects companies with senior safety experts. A client
enters a company name; the AI pipeline crawls public sustainability
reports, extracts EHS KPIs, benchmarks the company against industry
peers, and generates a structured proposal with a ranked shortlist of
three matched experts — before any money changes hands. The client
purchases a fixed-price engagement in one click, the expert delivers
the assessment, and the final report is surfaced inside the client
project workspace. The initial product targets EHS managers, COOs,
and compliance officers at mid-market and enterprise companies operating
in regulated industries.

## Goals

1. Remove expert-discovery friction by replacing manual outreach with
   an AI matchmaking pipeline that maps a company's specific risk profile
   to expert competency tags automatically.
2. Remove scoping uncertainty by generating a structured, benchmarked
   proposal with a visible price before the client commits to a purchase.
3. Ship a self-serve checkout flow for the three standard packages (CHF
   2,000 / 5,000 / 10,000) via Stripe, with Swiss MWST handled
   automatically through Stripe Tax.
4. Establish a verified expert network with an admin-gated approval step
   so only qualified senior EHS professionals appear in the directory and
   pipeline results.
5. Satisfy EU data residency and Swiss tax compliance from day one —
   these are non-negotiable launch requirements, not post-launch hardening.
6. Deliver a project workspace where the engagement lifecycle (kickoff,
   proposal, assessment report) lives in one place after purchase.

## Core User Flow

1. Client registers with a magic link or Google OAuth and lands in the
   portal.
2. Client enters a company name to trigger the AI pipeline.
3. The pipeline runs as a Trigger.dev background task: Claude Sonnet 4.6
   searches public sources for EHS news and sustainability reports,
   extracts 13 named KPIs, benchmarks them against industry peers, scores
   experts by competency tag match, and renders a proposal PDF.
4. An admin reviews the AI-generated proposal before it is released to
   the client (editorial gate — unreviewed proposals are never shared
   externally).
5. Client reviews the benchmarked proposal and selects a package;
   Stripe Checkout processes the payment and the webhook opens the
   engagement.
6. Expert is notified; the platform emails both parties' contact details
   so they can schedule kickoff via their own tools.
7. Expert delivers the assessment; the final report is uploaded to the
   project vault and surfaced in the client dashboard.
8. Client leaves feedback at engagement close.

## Features

### AI Intelligence Pipeline

- Company name → public EHS KPI extraction (FFR, TRI, TRIFR, LTI,
  LTIFR, HPRIs, fatalities, and 6 additional fields) via Claude with
  `web_search`, targeting GRI 403-9 sustainability reports.
- Industry peer benchmarking with monetized risk exposure (CHF cost
  factors for fatalities, LTIs, and HPRIs) producing a Total Opportunity
  Value figure.
- Expert matchmaking scores the full approved-expert roster against the
  company's risk profile using competency tags; outputs a ranked top-3
  shortlist.
- Proposal PDF generated by Claude from benchmark data + expert
  shortlist + selected package context; rendered via React-PDF and stored
  in Supabase Storage.
- Per-user daily run quota (default: 3/day) and monthly benchmark
  caching by `(company_slug, yyyy-mm)` to avoid redundant AI calls.
- Admin-managed EHS knowledge vault (PDFs: ISO 45001, sector guides,
  compliance frameworks) grounding the pipeline at extraction and
  proposal stages.

### Expert Marketplace

- Multi-step expert application: bio, competency tags, photo, optional
  CV upload.
- Admin approval gate before profiles are published.
- Public directory with search and category filter (no faceted filters
  at launch); each expert card links to a public profile page with a
  Contact dialog.
- Competency tags are the input to pipeline matchmaking.

### Authentication and Roles

- Clients: magic link (email OTP) or Google OAuth. No password option.
- Experts: Google OAuth only. No magic link. LinkedIn OAuth deferred to v2.
- Admin: env-var email whitelist gates access; no admins table at launch.
- Supabase row-level security is the canonical access layer — not
  application-level checks.
- Two portal surfaces: client project workspace and expert delivery
  dashboard. Public marketing site and expert directory are unauthenticated.

### Packages and Payments

- Four fixed-price tiers: EHS Snapshot (CHF 2,000 / 1 remote day),
  EHS Reality Check (CHF 5,000 / 2 on-site days), EHS Transformation
  Plan (CHF 10,000 / 5 on-site days), EHS Execution Partner
  (CHF 10,000 + CHF 1,850/day ongoing).
- Packages 1–3 via Stripe Checkout; package 4 via contact form
  → manual scoping.
- Stripe webhook writes to `orders` table + sends confirmation email
  via Resend.
- Stripe Tax applies Swiss MWST automatically. Live mode deferred until
  the Swiss legal entity is activated.

### Client Project Workspace

- Project board showing payment history and live engagement status.
- Document vault (signed URLs, scoped per engagement, EU-resident
  Supabase Storage).
- Final report library for completed assessments.
- Feedback collection at engagement close.
- In-platform scheduling deferred to v2; platform emails contact details
  at Confirmed state.

### Marketing Site

- 5-page public Next.js site (same codebase under `app/(marketing)`):
  Home, How It Works, Expert Network, Packages, Contact.
- English only. Production domain: `sme24.ch`.
- Contact form routes to the leads inbox via Resend.
- SEO basics and Organisation schema markup.

## Scope

### In Scope

- Next.js 15 + TypeScript full-stack app on Vercel.
- Supabase Auth, Postgres, and Storage for identity, relational data,
  and binary artifacts.
- Trigger.dev durable background tasks for all AI pipeline work.
- Anthropic Claude Sonnet 4.6 for research, KPI extraction, and proposal
  authoring.
- Stripe Checkout and webhook integration with Swiss MWST via Stripe Tax.
- Resend for transactional email (post-purchase confirmation, contact
  form intake).
- Sentry for runtime error monitoring across app and pipeline.
- Public marketing site (5 pages) and authenticated portal.
- Expert application, admin approval flow, public directory, and
  public profile pages.
- AI pipeline: crawler, KPI extraction, benchmarking, matchmaking,
  proposal generation, editorial review gate.
- Admin knowledge vault and proposal review console.
- Client project workspace: project board, document vault, report
  library, feedback collection.
- EU data residency and Swiss tax compliance.
- End-to-end AI pipeline stress test.

### Out of Scope

- Faceted filters on the expert directory.
- Company contact and decider-extraction module (Module 03 — deferred,
  not permanently excluded).
- Purchasing and contracting module, time-tracking, quality management.
- Saved search history and bookmarked companies.
- Live-mode Stripe before legal entity activation.
- Blog, content marketing, and careers pages.
- Full privacy policy and Impressum body copy.
- In-platform scheduling (v2).
- BI reporting dashboard (post-v1).
- EHS Vault / Bot for experts (post-v1, spec TBD by Philipp Hotz).
- Dark mode.
- Multi-language portal interface.
- Native mobile application.
- Enterprise security/compliance certifications (ISO, SOC-2).
- Ongoing maintenance post-launch.
- LinkedIn OAuth for experts (deferred to v2).
- Bilingual interface and `copy/de` translations (English only at launch).

## Success Criteria

1. `npm run lint` and `npm run build` pass with no TypeScript or lint
   errors on the current branch.
2. An authenticated client can enter a company name, trigger the full
   AI pipeline, and receive a reviewed proposal PDF — end to end.
3. Stripe Checkout completes for packages 1–3 and the Stripe webhook
   opens the project workspace without manual intervention.
4. An expert applicant can submit the multi-step form; an admin can
   approve or reject the application and the profile publishes immediately
   on approval.
5. All database writes enforce RLS; service-role access is restricted to
   Trigger.dev tasks and Stripe webhook handlers.
6. Customer data and generated artifacts reside in EU regions at all
   times.
7. Context documents contain concrete, implementation-ready guidance with
   no bracketed placeholder text before coding begins.
